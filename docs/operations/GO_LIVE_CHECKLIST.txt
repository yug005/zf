# ZER0FRICTION GO-LIVE CHECKLIST (2026 EDITION)

---

1. MISSING FEATURES (MUST-HAVES FOR 2026)
---------------------------------------

- MFA / WEBAUTHN (MULTI-FACTOR AUTH): Essential for any infra-related cloud service.
- TEAM/ORGANIZATION SUPPORT: Companies cannot use the product if they cannot share access among multiple developers.
- PUBLIC STATUS PAGES: A major marketing/trust feature for status monitoring tools.
- ADVANCED MONITORING TYPES: SSL Expiry checks, DNS validation, and gRPC support are the modern standard.
- SLACK/DISCORD INTEGRATION: Developers don't only use email. We need configured notification webhooks.

2. SECURITY VULNERABILITIES (GAPS TO CLOSE)
------------------------------------------

- REFRESH TOKEN ROTATION: Current refresh tokens are static. If a user's token is stolen, an attacker can maintain access indefinitely. Rotation prevents this.
- CSRF PROTECTION: Ensure that all POST/PUT/PATCH/DELETE endpoints have CSRF tokens to prevent browser-based attacks.
- PASSOWRD STRENGTH VALIDATION: Backend should reject weak passwords ("12345678", "password").
- RATE LIMITING/THROTTLING: Every endpoint needs a strict rate limit based on user tiers to avoid bill-scaling attacks or API abuse.
- SQL INJECTION (RAW QUERIES): While Prisma is safe, any raw query usage should be triple-checked for improper sanitization.

3. PROPOSED SOLUTIONS
--------------------

- SECURITY: Add '@nestjs/passport' WebAuthn strategies and implement 'Token Service' for rotation.
- TEAMS: Update 'User' model to have a 'Team' relation or a 'Workspace' model.
- NOTIFICATIONS: Integrate 'Slack Webhook URL' and 'Discord Webhook URL' into the Monitor alerts logic.
- INFRA: Setup 'Sentry' or 'OpenTelemetry' for backend error tracking.
